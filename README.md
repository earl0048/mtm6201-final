# MTM6201 Final Project

Final Project for 26W_MTM6201_020 Advanced Web Design with HTML and CSS at Algonquin College, within the Interactive Media Design program (Level 2).

Created by Ashlee Earle.

## Project Overview

This project is a fully responsive website developed to replicate a high-fidelity Figma design created in a previous course. The website represents a fictional activity and community centre and focuses on delivering a clean, engaging, and user-friendly experience across desktop, tablet, and mobile devices.

The original design process included:
- User and stakeholder research (community members, staff, vendors)
- Persona creation
- Competitive and feature research
- Mood board and style tile development
- Site mapping
- Wireframes and low-fidelity mockups
- High-fidelity mockups
- Interactive prototype in Figma

This project brings that design to a real github wesbite using HTML and CSS.

## Process

My process began with translating my Figma prototype into code while maintaining the intended visual identity and layout. I focused on building a responsive structure first, ensuring that the layout adapted properly across different screen sizes before refining visual details.

During development, I revisited my original design decisions and made improvements where necessary, particularly in accessibility. While the Figma prototype reflected the brand visually, some design choices, such as color contrast, were not optimal for readability. I adjusted elements like the navigation and footer colors to improve contrast while still preserving the overall brand aesthetic.

In addition to improving accessibility, I also updated several visual assets. The original Figma design used placeholder imagery, but during development I sourced higher-quality photos that better aligned with the brand’s color palette and overall aesthetic. This helped create a more cohesive and polished final product while improving the overall user experience.

## Challenges & Solutions

One of the main challenges I faced was getting the hero card sections to behave consistently across different screen sizes. Positioning, spacing, and responsiveness required multiple iterations and testing to achieve the intended look without breaking the layout.

To overcome this, I:
- Broke the problem down into smaller layout components
- Tested changes incrementally on the "go live" feature in vs code
- Adjusted padding, positioning, and container behavior
- Used responsive design techniques to ensure consistency across breakpoints

Another challenge was balancing design accuracy with accessibility. Some original color choices in the Figma design made text difficult to read, especially for users with visual impairments. I addressed this by updating certain colors, such as using darker backgrounds for navigation and footer areas, to improve contrast and readability while maintaining the brand’s look and feel.

## What I Learned

Through this project, I gained a deeper understanding of:
- Translating design prototypes into functional, responsive code
- The importance of accessibility in web design
- How small design decisions (like color contrast and spacing) impact usability
- Debugging layout issues and working through responsive challenges
- Structuring a project from concept to final implementation

I also learned that design is not always final—development is an opportunity to refine and improve the user experience beyond the original concept.

## Future Improvements

If I were to continue developing this project, I would enhance the hero sections on the Programs and Contact pages by incorporating relevant imagery. This would make the pages more visually engaging and help draw users in immediately.
